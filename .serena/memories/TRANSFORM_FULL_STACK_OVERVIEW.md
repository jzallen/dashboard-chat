# Transform Full Stack Overview

## Summary
Transforms are filter rules created from RAQB query builder JSON. They're stored in the database, applied during query execution via Ibis expressions, and persisted to transforms table. The lifecycle is: create → update status/conditions → soft-delete via status='deleted'.

```python
class Transform:
    id: str | None                          # UUID or None for unsaved
    name: str                               # Human-readable name
    condition_json: QueryBuilderJSON | None # RAQB JSON tree (value object)
    condition_sql: str | None = None        # SQL WHERE clause (for display/backwards compat)
    description: str | None = None
    status: TransformStatus = 'enabled'     # 'enabled' | 'disabled' | 'deleted'

    @property
    def is_enabled(self) -> bool:
        return self.status == 'enabled'

    def serialize(self) -> dict[str, Any]:
        """Serialize to JSON-compatible dict for HTTP responses."""
        return {
            'id': self.id,
            'name': self.name,
            'condition_json': dict(self.condition_json) if self.condition_json else None,
            'condition_sql': self.condition_sql,
            'description': self.description,
            'status': self.status,
        }

    # Dict-like access for SQLAlchemy bulk operations
    def __getitem__(self, key: str) -> object: ...
    def keys(self) -> list[str]: ...
    def __iter__(self): ...
```

### Key Points
- **Frozen dataclass**: Immutable (recommended for domain models)
- **condition_json**: RAQB JSON tree structure, converted via `QueryBuilderJSON.from_dict()`
- **condition_sql**: Kept for display/backwards compatibility (backend now generates via Ibis)
- **Status values**: 'enabled', 'disabled', 'deleted' (soft-delete via status change)
- **is_enabled property**: Returns True only if status == 'enabled'

### Repository Methods (`backend/app/repositories/metadata/repository.py`)

#### Create
```python
async def create_transforms_batch(
    self,
    dataset_id: str,
    transforms_input: list[dict[str, Any]],
) -> list[dict[str, Any]]:
    """Create multiple transforms in a single flush."""
    records = []
    for t in transforms_input:
        record = TransformRecord(
            dataset_id=dataset_id,
            name=t['name'],
            condition_json=t['condition_json'],
            condition_sql=t.get('condition_sql', ''),
            description=t.get('description'),
            nl_prompt=t.get('nl_prompt'),
        )
        self._session.add(record)
        records.append(record)
    
    await self._session.flush()
    for record in records:
        await self._session.refresh(record)
    
    return [self._transform_to_dict(r) for r in records]
```

#### Update
```python
async def update_transforms(self, updates: list[dict[str, Any]]) -> None:
    """Batch update transforms in a single query.
    
    Args:
        updates: List of dicts, each containing 'id' and fields to update.
    """
    from sqlalchemy import update
    
    if updates:
        await self._session.execute(update(TransformRecord), updates)
        await self._session.flush()
```

#### Find by SQL (for deduplication)
```python
async def find_transform_by_sql(
    self,
    dataset_id: str,
    condition_sql: str,
) -> dict[str, Any] | None:
    """Find an existing transform with matching SQL."""
    result = await self._session.execute(
        select(TransformRecord)
        .where(TransformRecord.dataset_id == dataset_id)
        .where(TransformRecord.condition_sql == condition_sql)
        .order_by(TransformRecord.created_at.asc())
        .limit(1)
    )
    transform = result.scalar_one_or_none()
    if not transform:
        return None
    return self._transform_to_dict(transform)
```

### Transform Application via Ibis (_build_table)

```python
def _build_table(self, table_name: str | None = None) -> ibis.Table:
    """Build Ibis table with filters applied.
    
    Reads schema from parquet in S3 when available, otherwise falls back
    to building a table expression from schema_config.
    """
    try:
        conn = self._get_connection()  # DuckDB with S3/MinIO config
        table = conn.read_parquet(self._s3_path(), table_name=table_name)
    except Exception:
        table = self._build_table_from_schema(table_name)
    
    # Select columns from schema
    fields = self.schema_config.get("fields", {})
    if fields:
        table = table.select(*fields.keys())
    
    # Apply active transform filters
    active_filters = [
        t.condition_json.as_ibis_filter(table)  # Convert RAQB to Ibis expression
        for t in self.transforms
        if t.is_enabled and t.condition_json
    ]
    if active_filters:
        table = table.filter(*active_filters)
    
    return table
```

```python
@property
def display_sql(self) -> SQLQuery:
    """Human-readable SQL with dataset name and alias."""
    try:
        table = self._build_table(table_name=self.name)
        sql = ibis.to_sql(table, dialect="duckdb", pretty=True)
    except Exception as e:
        return f"-- Error generating SQL: {str(e)}"
    
    # Post-processing to make it human-readable
    alias = self._table_alias()
    sql = sql.replace('"t0"', alias)
    sql = re.sub(rf'{alias}\."(\w+)"', rf'{alias}.\1', sql)
    
    # Expand SELECT * to explicit columns
    if "SELECT\n  *\n" in sql:
        fields = self.schema_config.get("fields", {})
        if fields:
            col_list = ",\n  ".join(f"{alias}.{col}" for col in fields.keys())
            sql = sql.replace("SELECT\n  *\n", f"SELECT\n  {col_list}\n")
    
    return sql
```

## 6. BACKEND QUERYBUILDER JSON TYPE (`backend/app/types.py`)

**Location**: `/workspaces/dashboard-chat/backend/app/types.py`

### QueryBuilderJSON Class

```python
class QueryBuilderJSON(dict):
    """Value object for RAQB query builder JSON.
    
    Extends dict so it serializes naturally with asdict() and JSON serializers.
    Provides conversion to Ibis expressions.
    """

    @classmethod
    def from_dict(cls, d: dict[str, Any] | None) -> QueryBuilderJSON | None:
        """Create from dictionary, returning None if empty."""
        if not d:
            return None
        return cls(d)

    def as_ibis_filter(self, table: ibis.Table) -> ibis.expr.types.BooleanValue:
        """Convert to Ibis filter expression."""
        return self._process_group(self, table)

    def _process_group(self, group: dict, table: ibis.Table) -> ibis.expr.types.BooleanValue:
        """Process a group node (contains AND/OR conjunction)."""
        children = group.get("children1", {})
        props = group.get("properties", {})
        conjunction = props.get("conjunction", "AND")
        is_negated = props.get("not", False)
        
        if not children:
            return ibis.literal(True)
        
        filters = [self._process_node(child, table) for child in children.values()]
        filters = [f for f in filters if f is not None]
        
        if not filters:
            return ibis.literal(True)
        
        # Reduce with conjunction (AND = &, OR = |)
        combine = (lambda a, b: a | b) if conjunction == "OR" else (lambda a, b: a & b)
        result = reduce(combine, filters)
        
        return ~result if is_negated else result

    def _process_rule(self, rule: dict, table: ibis.Table) -> ibis.expr.types.BooleanValue | None:
        """Process a rule node (single condition)."""
        props = rule.get("properties", {})
        field, operator = props.get("field"), props.get("operator")
        values = props.get("value", [])
        
        if not field or not operator:
            return None
        
        column = table[field]
        value = values[0] if values else None
        
        match operator:
            case "equal" | "select_equals":
                return column == value
            case "not_equal" | "select_not_equals":
                return column != value
            case "less":
                return column < value
            case "less_or_equal":
                return column <= value
            case "greater":
                return column > value
            case "greater_or_equal":
                return column >= value
            case "between" if len(values) >= 2:
                return column.between(values[0], values[1])
            case "not_between" if len(values) >= 2:
                return ~column.between(values[0], values[1])
            case "like":
                return column.like(f"%{value}%")
            case "not_like":
                return ~column.like(f"%{value}%")
            case "starts_with":
                return column.like(f"{value}%")
            case "ends_with":
                return column.like(f"%{value}")
            case "is_null":
                return column.isnull()
            case "is_not_null":
                return column.notnull()
            case "is_empty":
                return column.isnull() | (column == "")
            case "is_not_empty":
                return column.notnull() & (column != "")
            case "select_any_in":
                return column.isin(values)
            case "select_not_any_in":
                return ~column.isin(values)
            case _:
                return None
```

### Supported Operators
- **Comparison**: equal, not_equal, less, less_or_equal, greater, greater_or_equal
- **Range**: between, not_between
- **Text**: like, not_like, starts_with, ends_with
- **Nullability**: is_null, is_not_null
- **Emptiness**: is_empty, is_not_empty
- **Set operations**: select_any_in, select_not_any_in

```typescript
function performToolAction(
  name: string,
  args: Record<string, unknown>,
  handlers: ToolCallHandlers
): void {
  switch (name) {
    case "filterTable": {
      // Add single filter (compound via column-level OR)
      const { column, operator, value } = args;
      handlers.setColumnFilters((prev) => {
        const existing = prev.find((f) => f.id === column);
        const newCondition = { operator, value };
        
        if (existing) {
          // Merge into compound filter (AND logic)
          const conditions = (existing.value as any).conditions
            ? [...(existing.value as any).conditions, newCondition]
            : [{ ...(existing.value as any), transformId: (existing.value as any).transformId }, newCondition];
          return prev.map((f) => f.id === column ? { ...f, value: { conditions } } : f);
        }
        return [...prev, { id: column, value: newCondition }];
      });
      break;
    }

    case "replaceColumnFilter": {
      // Replace all filters on a column with new conditions
      const { column, filters } = args;
      handlers.setColumnFilters((prev) => {
        const withoutColumn = prev.filter((f) => f.id !== column);
        if (!filters || filters.length === 0) return withoutColumn;
        if (filters.length === 1)
          return [...withoutColumn, { id: column, value: filters[0] }];
        return [...withoutColumn, { id: column, value: { conditions: filters } }];
      });
      break;
    }

    case "generateFilter": {
      // Apply RAQB tree (replace all existing filters)
      const { raqb_tree } = args;
      const filters = raqbTreeToFilters(raqb_tree);
      handlers.setColumnFilters(filters);
      break;
    }

    case "sortTable": {...}
    case "addRow": {...}
    case "deleteRow": {...}
    case "clearFilters": {...}
    case "clearSort": {...}
  }
}
```

### RAQB Tree to Filters Conversion

```typescript
function raqbTreeToFilters(tree: RaqbGroup): Array<{ id: string; value: unknown }> {
  const filterMap = new Map<string, Array<{ operator: string; value: unknown }>>();
  
  for (const child of Object.values(tree.children1)) {
    if (child.type === "rule") {
      const field = child.properties.field;
      const op = mapRaqbOperator(child.properties.operator);
      const val = child.properties.value[0];
      if (!filterMap.has(field)) filterMap.set(field, []);
      filterMap.get(field)!.push({ operator: op, value: val });
    }
  }
  
  return Array.from(filterMap.entries()).map(([field, conditions]) =>
    conditions.length === 1
      ? { id: field, value: conditions[0] }
      : { id: field, value: { conditions } }
  );
}

function mapRaqbOperator(op: string): string {
  switch (op) {
    case "equal": return "equals";
    case "not_equal": return "notEquals";
    case "greater": return "gt";
    case "less": return "lt";
    case "greater_or_equal": return "gte";
    case "less_or_equal": return "lte";
    case "like": return "contains";
    default: return op;
  }
}
```

```typescript
export function getSystemPrompt(tableSchema: TableSchema): string {
  const columnDescriptions = tableSchema.columns
    .map((c) => {
      const base = `  - "${c.id}" (${c.type})`;
      const profileStr = formatProfile(c.profile);
      return profileStr ? `${base} -- ${profileStr}` : base;

    .join("\n");

  const activeFilterLines =
    tableSchema.activeFilters && tableSchema.activeFilters.length > 0
      ? `\nACTIVE FILTERS:\n${tableSchema.activeFilters.map((f) => `  - ${f.column} ${f.operator} ${f.value}`).join("\n")}\n`
      : "\nNo active filters.\n";

  return `You are a helpful assistant that controls a data table. You can filter, sort, add rows, and delete rows using the provided tools.

CURRENT TABLE SCHEMA:
${columnDescriptions}

Total rows: ${tableSchema.rowCount}
${activeFilterLines}
When the user mentions a value that matches a known column value from the profile above, use it for exact filtering with the "equals" operator.

INSTRUCTIONS:
1. To ADD a new filter, use "filterTable"
2. To CHANGE an existing filter, use "replaceColumnFilter" (NOT filterTable)
3. For sorting, use "sortTable"
4. For adding rows, use "addRow"
5. For deleting rows, use "deleteRow"
6. Use "clearFilters" or "clearSort" to reset

Be concise. Confirm what action you're taking.`;
```

## KEY DESIGN PATTERNS

### 1. **Transform Lifecycle**
- **Create**: Frontend sends condition_json + condition_sql → Backend persists to DB
- **Read**: Backend fetches transforms with dataset, converts to domain objects
- **Update**: Batch update via PATCH (name, description, condition, status)
- **Delete**: Soft-delete via status='deleted' (not hard-deleted)

### 2. **RAQB JSON → SQL Conversion**
- **Frontend**: Generates condition_sql from condition_json (not currently done)
- **Backend**: Stores both, regenerates SQL via Ibis on query execution
- **Forwards compatibility**: Stores both for display/backwards compat

### 3. **Transform Status States**
- **'enabled'**: Active (applied in queries via _build_table)
- **'disabled'**: Inactive (stored but not applied)
- **'deleted'**: Soft-deleted (filtered out on reads, not hard-deleted)

### 4. **Filtering in Queries**
```python
# In _build_table:
active_filters = [
    t.condition_json.as_ibis_filter(table)
    for t in self.transforms
    if t.is_enabled and t.condition_json
]
if active_filters:
    table = table.filter(*active_filters)
```

### 5. **Batch Operations**
- Both create and update are batch operations
- Batch is atomic (flush once, all or nothing)
- Events published after successful persist

### 6. **Event Sourcing via Outbox**
- Transforms created/updated events are written to outbox table
- Async event handler processes them (e.g., for audit, external systems)

### 7. **Authorization Check**
```python
# In transform use cases:
await _verify_dataset_access(metadata_repo, dataset_id)
# Checks: dataset exists + user's org owns parent project
```

## DATA CLEANING TRANSFORMS (v2 Extension)

### New Transform Types
- **filter** (existing): Row-level WHERE predicates via condition_json/condition_sql
- **clean**: Column-level cleaning (trim, case, fill_null) via expression_config/expression_sql
- **alias**: Column rename operations via expression_config (RENAME stage)
- **map**: Value mapping (CASE WHEN) via expression_config/expression_sql

### New Fields on Transform Model
- `transform_type: TransformType = 'filter'` — Literal['filter', 'clean', 'alias', 'map']
- `target_column: str | None = None` — Target column for clean/alias/map
- `expression_sql: str | None = None` — Server-generated display SQL
- `expression_config: dict | None = None` — Structured JSON config
- `created_at: datetime | None = None` — For ordering cleaning transforms

### CleaningExpression Class (backend/app/types.py)
Converts expression_config JSON → Ibis column expressions:
- `trim` → `col.strip()` → `TRIM(column)`
- `case upper/lower/title` → `col.upper()/lower()/capitalize()` → `UPPER/LOWER/INITCAP(column)`
- `fill_null` → `col.fill_null(value)` → `COALESCE(column, value)`
- `map_values` → `ibis.case().when(...)` → `CASE WHEN ... END`
- `alias` → handled in RENAME stage → `column AS "alias"`

### Three-Stage Pipeline (_build_table)
1. **MUTATE** — Apply clean/map transforms sorted by created_at via `.mutate()`
2. **FILTER** — Apply filter transforms via `.filter()` (existing behavior)
3. **RENAME** — Apply alias transforms via `.rename()`

### Cross-Field Validation (TransformCreate schema)
- filter: requires condition_json + condition_sql; rejects expression fields
- clean/alias/map: requires target_column + expression_config; rejects condition fields
- Default transform_type = 'filter' for backward compatibility

### 8 New Chat Tools (shared/chat/prompts.ts)
trimWhitespace, standardizeCase, renameColumn, fillNulls, mapValues, applyCleaningTransform, undoCleaningTransform, reEnableCleaningTransform

### Extended TableSchema (shared/chat/types.ts)
- Column: added `alias?: string`
- Schema: added `activeCleaningTransforms?: Array<{id, column, operation, details?}>`
