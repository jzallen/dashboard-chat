# US-204: Cold deep-link to a project resolves active_scope before paint (and surfaces a named-diagnostic for cross-tenant / stale links)

> **Wave**: DISCUSS — `project-and-chat-session-management` (J-002)
> **Date**: 2026-05-13
> **Persona**: Maya Chen (clicking a bookmark or shared link)
> **Job mapped**: J002-Job-3 (Open a deep link cold)
> **Slice**: Slice 1 — Walking skeleton (happy path) + Slice 1
> (failure path — same machine logic; same slice)
> **Anticipated machine state names**: `resolving_initial_scope`,
> `project_selected`, `scope_mismatch_terminal`

## Problem

Deep links to project URLs are a common access pattern: bookmarks,
links shared in Slack, links pasted into email. Today, the cold-load
behavior of `/projects/{projectId}` is "render the app shell, fetch
the project, then render the body" — a 200-500ms flicker between
the URL navigation and the project chip appearing. Worse: if the URL
is stale (project deleted) or cross-tenant (a link to a project the
user lost access to), the failure path is unpredictable — sometimes
a blank page, sometimes a raw 403, sometimes a redirect to a generic
"Projects" grid with no explanation.

This is part of the canonical bug class the J-002 machine retires
(per `adr-027:14`). For deep links specifically, the ScopeResolver
invariant 4 (per ADR-029 §1) commits to **returning 403 with named
diagnostic for cross-tenant project access**. The user-facing
behavior is what this story commits to.

## Who

* **Maya Chen (deep-link visitor)** — opens a URL like
  `/projects/q4-analytics` in a cold tab or after a clipboard paste.
  Wants the chip and the body to paint together with the right
  project, OR a clear error state if the link is dead.
* **Maya Chen (cross-tenant accident)** — clicks a Slack link a
  colleague sent. The colleague is in a different org; Maya cannot
  access that project. Wants a clear "this project is not
  accessible" message, not a confusing redirect or a 500 page.

## Solution

J-002 enters `resolving_initial_scope` on every cold load (entry
from J-001 `ready` AND every navigation into a project URL). The
machine reads `intent_project_id` from the URL params (via the
Remix loader at `app/routes/projects.$projectId.tsx`) and passes
it to the ScopeResolver.

The ScopeResolver (per ADR-029 §1):

* **Invariant 1**: `active_scope.org_id` equals the JWT's `org_id`
  claim.
* **Invariant 4**: cross-tenant access → 403 with diagnostic
  `scope mismatch: project belongs to a different org`.
* **Invariant 5**: if URL says project A but the user's machine
  context (e.g., from a prior tab) said project B, the URL wins for
  cold deep-links; `scope_reconciled` is emitted for observability.

**Happy path**: the resolver returns the authoritative `active_scope`
with `project_id` populated. J-002 transitions to `project_selected`
BEFORE the page body renders (the loader's return value drives the
render).

**Failure path**: the resolver returns 403 with the diagnostic.
J-002 transitions to `scope_mismatch_terminal`. The route renders
the named-diagnostic panel (Scene 6 from the visual).

## Elevator Pitch

* **Before**: Maya pastes a project URL into a fresh tab → app shell
  renders blank → fetch project → project chip flickers in →
  page body renders. ~500ms of "is this the right project?"
* **After**: Maya pastes a project URL into a fresh tab → Remix
  loader runs → J-002 resolves `active_scope.project_id` → page
  renders with project chip + body together. <300ms; no flicker.
* **Decision enabled (happy path)**: Maya trusts that the link is
  the source of truth. Bookmarks and shared links are now
  first-class entry points.
* **Decision enabled (failure path)**: Maya knows immediately
  whether the link is dead and what to do next ("Back to projects").
  She doesn't waste time trying to interpret a blank page.

## Domain examples

### 1: Happy path — cold deep link to "Q4 Analytics"

Maya bookmarked `/projects/q4-analytics` last week. This morning
she opens it cold. The Remix loader at
`app/routes/projects.$projectId.tsx` runs server-side. It calls
J-002's projection endpoint with `intent_project_id = "q4-analytics"`.
The ScopeResolver validates: JWT carries `org_id` for Acme Data;
"q4-analytics" belongs to Acme Data; access OK. The projection
returns with `active_scope.project_id = "q4-analytics"`. J-002 is
in `project_selected`. The route renders with the project chip
and the project's body together.

### 2: Cross-tenant — Slack link Maya can't access

Maya clicks a Slack link `/projects/strategic` shared by a
colleague at Other Org Inc. Maya is in Acme Data. The ScopeResolver
runs: JWT's `org_id` is for Acme Data; "strategic" belongs to
Other Org Inc; → 403 with `scope mismatch: project belongs to a
different org`. J-002 transitions to `scope_mismatch_terminal`. The
route renders Scene 6's panel: "This project is no longer
accessible" + correlation_id + "Back to projects" CTA.

### 3: Stale link — project deleted since the link was shared

Maya clicks a link to a project that was deleted yesterday. The
ScopeResolver runs: JWT carries Acme Data org_id; project not
found in the org. → 404 (mapped to the same `scope_mismatch_terminal`
state for UX simplicity). The panel reads "This project is no
longer accessible" with copy variant tag
`underlying_cause_tag = "project_not_found"`.

### 4: Stale link — Maya lost access (membership revoked)

Maya was removed from a project she previously had access to (a
future surface; the current backend doesn't support per-project
membership, but the ScopeResolver will when it does). Same UX as
cross-tenant: `scope_mismatch_terminal` panel with copy variant
tag `underlying_cause_tag = "access_revoked"`.

### 5: Deep link with intent_session_id and intent_dataset_id

Maya bookmarked `/projects/q4-analytics/datasets/sales_2026` per
`frontend/app/routes.ts:24-27` (the project-dataset-detail route).
The Remix loader passes `intent_project_id`, `intent_resource_type
= "dataset"`, `intent_resource_id = "sales_2026"`. The ScopeResolver
validates all three; J-002 transitions to `project_selected` AND
the route renders with the dataset chip in the gutter too.

## Acceptance Criteria

* [ ] On a cold deep-link to `/projects/{projectId}` (or any
  project-scoped route), the Remix loader runs server-side and
  passes `intent_project_id` to J-002's projection.
* [ ] The ScopeResolver enforces ADR-029 §1 invariants 1+4 before
  the route renders: JWT-claim parity (1) + cross-tenant rejection
  (4).
* [ ] On happy path, J-002 transitions to `project_selected` and the
  project chip + body paint together; first-paint latency ≤300ms at
  p95 (the budget per ADR-029 invariant 4 — pure function over
  already-loaded state).
* [ ] On cross-tenant or project-not-found, J-002 transitions to
  `scope_mismatch_terminal` with `underlying_cause_tag` populated
  (closed-vocabulary discriminated union per
  `docs/evolution/2026-05-12-user-flow-state-machines/discuss/user-stories.md`
  US-003 pattern).
* [ ] The named-diagnostic panel displays the `correlation_id` of
  the originating request; no raw 403 / 404 / 500 is visible.
* [ ] "Back to projects" CTA from `scope_mismatch_terminal` re-enters
  `resolving_initial_scope` with intent cleared.
* [ ] Deep links carrying `intent_resource_id` (e.g.,
  `/projects/{id}/datasets/{datasetId}`) materialize `active_scope`
  with both `project_id` AND `resource_*` on first paint.
* [ ] The TS harness exposes `harness.j002.open_deep_link({...})`
  and `harness.j002.assert_scope_mismatch({underlying_cause_tag})`.

## Outcome KPIs

* **Who**: Users entering via a cold deep link (bookmark, shared
  link).
* **Does what**: Sees the project chip + body painted together
  with the right project at <300ms (p95), OR a named-diagnostic
  panel within 300ms if the link is dead/cross-tenant.
* **By how much**: 100% of valid deep-links resolve in <300ms p95;
  100% of invalid deep-links surface the named-diagnostic panel
  (NOT a blank page or raw error) at p99.
* **Measured by**: FE event
  `j002_deep_link_resolved { project_id, latency_ms, outcome:
  "happy" | "cross_tenant" | "project_not_found" | "access_revoked" }`.
* **Baseline**: Today, deep-link cold-loads observe ~200-500ms of
  flicker; ~10% of stale links land on a blank page (developer
  recall; uninstrumented).

## Technical notes

* Depends on the Remix loader at
  `app/routes/projects.$projectId.tsx` reading the J-002 projection
  with `intent_project_id` from URL params (per ADR-029 §2 Option
  D example at lines 82-93).
* Depends on the ScopeResolver returning structured diagnostics on
  403 (per ADR-029 §1 invariant 4 — diagnostic body is the
  closed-vocabulary tag).
* The `scope_reconciled` FlowEvent (ADR-029 §1 invariant 5) is
  out-of-scope for THIS story (no UX implication; observability
  only) but its emission is required as part of the contract.
