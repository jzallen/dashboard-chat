# Evolution

Post-mortem / lessons-learned summaries for completed nwave-ai features,
produced by `/nw-finalize`. Each entry archives the feature's wave
artifacts and surfaces lasting decisions, deferred items, and lessons.

Earlier entries are flat single-file summaries
(`YYYY-MM-DD-{feature-id}.md`). Starting with
`2026-05-12-user-flow-state-machines/` the layout is a directory with
`FINALIZE.md` plus a verbatim copy of the feature's `discuss/`,
`design/`, `distill/`, `deliver/` subtree.

## Index

- [2026-07-23-ui-state-hexagonal-transport/](2026-07-23-ui-state-hexagonal-transport/FINALIZE.md) — **precursor, subsumed by ADR-044 + ADR-046.** ADR-040's six-LEAF hexagonal-transport migration of the `ui-state` flow transport (parameterized `/flow/:machine/*` seam + ~1939-line orchestrator → `FlowStrategy` port + generic pump + per-machine routers + a hard swap to a server-authoritative `SettledStateStore`). **LEAF-1/2/3 delivered** 2026-05-16 → 05-18 (registry + alias router + orchestrator-pump carve), each with a per-marker A/B behavior-neutrality proof (Δ=0). **LEAF-4/5/6 were never delivered as ADR-040 LEAFs** — a ratified pivot overtook the plan: ADR-044 (ChatApp coordinator, 2026-05-25) replaced the `FlowOrchestrator` and its Phase-4 delete (`3d810121`) removed the very `orchestrator.ts` + `strategy.ts` scaffold LEAF-1/3 built, and ADR-046 (StateProxy `/state` surface, 2026-05-29) realized LEAF-5's store-model exit (`deriveStateDocument` + migration-equivalence gate) and LEAF-6's alias/per-machine-surface removal (`a3cb2eb6`) under a clean cutover the "nothing in production" FE-rewrite made cheaper. Nothing the LEAFs *built* survives; three of ADR-040's *ideas* do — header-derived identity (verbatim in ADR-046), canonical-name dispatch (folded into ADR-039), and the store-model exit. Lessons: a behavior-neutral refactor mid-flight is hostage to the strategy above it; a pivot can make elaborate safety machinery obsolete, not merely unfinished.
- [2026-07-22-ssr-ui-server-gateway/](2026-07-22-ssr-ui-server-gateway/FINALIZE.md) — the SSR **ui-server** (backend-for-frontend) gateway: `ui/` gets one integration point — its own RRv7 server-side runtime (`/ui-server/*` loaders/actions/resource routes) that brokers the backend/agent surfaces server-side instead of the browser fanning out to `/api`, `/worker`, `/ui-state`. Shipped as a strangler-fig migration through the `DataCatalog` `dataSource` seam across `catalog-behind-bff` Releases 1–6 (S1 server-side `/api` client → S2/S3 reads on server loaders → mutations on RRv7 actions → S6 upload saga → S7 onboarding driver → DC-119 write-through convergence with **SSE as a pure `revalidator.revalidate()` trigger** → S8 retire `backendClient`), plus the DC-104 `bff→ui-server` rename and the DC-193 `web-ssr`/`reverse-proxy` image repoint. Ratifies ADR-033 (source-tree topology) + ADR-034 (RRv7 framework mode incl. the 2026-06-25 idiomatic-RRv7 amendment; closes upstream UI-1). **Deferred:** M2M on-behalf-of — the ui-server still forwards the user credential, so the Phase-4 "downstream trusts one caller" security bank is captured as intent, not yet delivered. Lessons: a dataSource seam turns migration into a repoint not a rewrite; sequence the stream relay last but the priority path first; roadmap `status` fields go stale — git is the source of truth.
- [2026-07-09-org-onboarding/](2026-07-09-org-onboarding/FINALIZE.md) — **precursor, subsumed by `client-driven-onboarding`.** The initial first-run onboarding cut (org + one default project), shipped 2026-06-10 as MR-1 (backend) + MR-2 (`ui/`). Its durable contribution is the **backend DEV-reachability affordance** — nullable `organizations.created_by` (migration 018), `created_by` stamping with the "My First Project" auto-create dropped (user-driven first project), and the `DEV_NO_ORG` DB-resolution flag that closes the infinite dev loop — all still live on `main`. Its onboarding *layer* (the `org_form_submitted`/`create_project_submitted` wire vocabulary, the `ui/` surfaces, and the `tests/acceptance/org-onboarding/` suite) was reworked in place by CDO (ADR-048/049/050) for the client-reported model, closing this feature's UI-1 misnomer structurally. Archived rather than finalized standalone; CDO's own FINALIZE will back-reference it. Lessons: theater tests hid a `POST /api/orgs` 500-on-success; an org-global catalog memo with no invalidation produced a stale post-onboarding shell the API seam couldn't see.
- [2026-05-29-chat-app-machine/](2026-05-29-chat-app-machine/FINALIZE.md) — ADR-044 complete: the imperative `FlowOrchestrator` (~1400 lines) is replaced by a declarative `ChatApp` XState v5 parent coordinator and then deleted. Gates green: backend 1348 pytest, ui-state 221 + frontend 604 vitest.
