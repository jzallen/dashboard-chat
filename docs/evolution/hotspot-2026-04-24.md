## Observations

### Critical refactor candidates

- **`backend/app/repositories/metadata/repository.py` (999 LOC, 22 commits)** — Near the 1000-line cliff. High churn × high LOC is the classic Tornhill "crime scene" pattern. Decompose into focused repositories (dataset, project, session, etc.). Entry point for `/nw-refactor` at L3/L4 (Extract Method / Move Responsibility).
- **`backend/app/controllers/http_controller.py` (527 LOC, 21 commits)** — Controllers should delegate, not accumulate. Likely violates SRP. Candidate for `nw-legacy-refactoring-ddd` extraction.
- **`backend/app/repositories/metadata/__init__.py` (373 LOC, 14 commits)** — 373 lines in an `__init__.py` is a smell. Package init files should re-export, not implement.
- **`backend/app/models/dataset.py` (318 LOC, 20 commits)** — Central domain model with sustained churn. Review for god-object drift.

### Family of similar files (candidates for consolidation)

The `backend/app/use_cases/sql_access/` directory has **10+ sibling use cases** (enable, disable, sync, regenerate, reconcile, start/stop/restart, get, get_status), each 60–131 LOC, each with 10–14 commits in the window. The uniform churn suggests parallel evolution — a candidate for a shared abstraction (state machine, strategy, or orchestrator) rather than 10 near-duplicate modules. Review with `/nw-refactor` or `/nw-mikado` for multi-module coordination.

### Test coverage gaps

- `create_dataset_from_upload.py` (172 LOC, 17 commits) and `upload_file.py` (126 LOC, 14 commits) are high-churn use cases without a matching high-churn test file in the top 20. Worth confirming coverage before further changes.

### Low-signal churn (expected)

- `package-lock.json`, `backend/uv.lock`, `CHANGELOG.md`, `pyproject.toml`, `package.json` — auto-generated or per-release churn. No action beyond normal monitoring.
- `docker-compose.yml`, `backend/app/main.py`, `backend/app/config.py`, `reverse-proxy/App.tsx` — infrastructure / composition roots. Churn is expected; action only if coupling or complexity grows.
