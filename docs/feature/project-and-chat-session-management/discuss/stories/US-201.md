# US-201: First-time-in-org user lands in the no-projects empty state

> **Wave**: DISCUSS ŌĆö `project-and-chat-session-management` (J-002)
> **Date**: 2026-05-13
> **Persona**: Maya Chen (just after J-001 / first-time-in-org)
> **Job mapped**: J002-Job-1 (Resume ŌĆö degraded form, since Maya has
> no project to resume yet) + the implicit J-002 invariant that
> entry into J-002 always settles in a coherent state
> **Slice**: Slice 1 ŌĆö Walking skeleton
> **Anticipated machine state names** (informational only):
> `resolving_initial_scope`, `no_projects_empty_state`,
> `creating_project`, `project_selected`

JWT carries org_id; state machine is server-owned in the `ui-state/`

## Problem

Maya just completed J-001 ŌĆö her org "Acme Data" was created
2026-05-13 and the JWT in her browser carries `org_id` for "Acme
Data". J-001 transitioned to `ready` with `session.current` set to a
pre-created placeholder session that points at the **org**, not at a
project (per J-001 ┬¦"ready" integration_checkpoint: "Three machine
fields are populated: user.display_name, org.name, session.current").

But Maya has zero projects in "Acme Data" ŌĆö the org was just created.
Today, after J-001 `ready`, the FE app shell renders with a blank
main pane and no clear next step. The user is dropped into ambiguity:
"I'm signed in, but where am I supposed to go?"

The path forward exists (the user can click "+ New Project" in the
nav per `frontend/app/routes.ts:21`) but is **not foregrounded**.
The friction is small but real, and the empty state is the first
post-sign-in moment that should feel like onboarding, not orphaning.

## Who

* **Maya Chen (first-time-in-org)** ŌĆö completed J-001 within the
  last minute; has a fresh JWT with `org_id` populated; has zero
  projects in her new org; wants to feel guided into creating her
  first project rather than dropped into an ambiguous app shell.

## Solution

J-002 enters `resolving_initial_scope` on transition from J-001
`ready`. It reads the project list via
`backend/app/use_cases/project/list_projects.py:17-27` and, on
finding zero projects in the user's org, transitions to
`no_projects_empty_state`. The FE renders a welcoming empty-state
panel with a "Create your first project" CTA. The CTA triggers a
single-step `POST /api/projects` call; J-002 transitions through
`creating_project` to `project_selected` for the new project.

## Elevator Pitch

* **Before**: After J-001, Maya signs in and the app shell renders
  with the org chip and a blank main pane. The "+ New Project"
  affordance is in the nav but not foregrounded; Maya is unsure what
  to click first.
* **After**: After J-001, Maya signs in and lands on a welcome panel
  at `/` (the app-shell root) that reads "Welcome to Acme Data,
  Maya! Let's get started by creating your first project." with a
  project-name input and a "Create project" CTA. Typing a name and
  clicking Create lands her in `project_selected` for the new project
  with the project chip painted.
* **Decision enabled**: Maya decides what to name her first project
  (an act of commitment that says "I'm using this tool") rather than
  pausing to scan the nav.

## Domain examples

### 1: Happy path ŌĆö Maya creates her first project

After J-001 `ready`, J-002 enters `resolving_initial_scope` and
calls `list_projects(user=maya)`. The response is `{items: [],
next_cursor: null, has_more: false}`. J-002 transitions to
`no_projects_empty_state`. The FE renders the welcome panel. Maya
types "Q4 Analytics" into the input and clicks "Create project".
J-002 transitions to `creating_project`; the `POST /api/projects`
call succeeds with the new project_id; J-002 transitions to
`project_selected` for "Q4 Analytics" with `active_scope.project_id`
populated. The app shell paints the project chip.

### 2: Edge ŌĆö Backend transient failure on create_project

Maya types "Q4 Analytics" and clicks "Create project". The backend
returns 503 (transient). J-002 transitions to `error_recoverable`
with `underlying_cause_tag = "transient"` and the `correlation_id`
of the create attempt. The UI shows "We couldn't create your project
right now. Try again." with a "Try again" CTA. Maya clicks it; J-002
re-enters `creating_project` with the SAME correlation_id; the retry
succeeds.

### 3: Boundary ŌĆö Project name validation failure

Maya types an empty name and clicks "Create project". The backend
returns 400 with a validation error. J-002 stays in
`no_projects_empty_state` with an inline form error: "Please enter a
project name." No project is created.

## Acceptance Criteria

* [ ] On entry from J-001 `ready` with zero projects, J-002
  transitions through `resolving_initial_scope` to
  `no_projects_empty_state` within 300ms (p95).
* [ ] The `no_projects_empty_state` UI shows
  `"Welcome to ${org.name}, ${user.first_name}!"` with a
  project-name input and a "Create project" CTA. No suggestion
  chips appear in this state.
* [ ] Clicking "Create project" with a valid name transitions
  through `creating_project` to `project_selected`; the project
  chip paints with the new project's name; `active_scope.project_id`
  equals the new project's id.
* [ ] Empty project name returns the machine to
  `no_projects_empty_state` with an inline form error; no
  `POST /api/projects` is attempted.
* [ ] Transient `POST /api/projects` failure transitions to
  `error_recoverable` carrying the create-attempt's `correlation_id`;
  retry re-enters `creating_project` with the same id.
* [ ] The TS harness exposes `harness.j002.create_first_project(name)`
  composing with `harness.user_flow.begin_auth("maya-first-time")`.
* [ ] No FE component reads `project_id` from `useParams` outside a
  route loader (inherited ESLint rule from ADR-029 ┬¦2).

## Outcome KPIs

* **Who**: First-time-in-org users (just completed J-001 with zero
  projects).
* **Does what**: Reaches a foregrounded, named next step (the
  no-projects empty state) within 300ms of J-001 `ready` AND creates
  a first project within the same session.
* **By how much**: 100% reach the empty state with no dead-end
  shell paint; Ōēź80% create a first project before navigating away.
* **Measured by**: FE event `no_projects_empty_state_reached` +
  `first_project_created`; conversion ratio per session.
* **Baseline**: Today, ~30% of first-time users navigate away
  without creating a project (developer recall; uninstrumented).

## Technical notes

* Depends on J-001 `ready` being reached AND the J-002 machine
  being registered in the orchestrator (`ui-state/index.ts`).
* Depends on `backend/app/use_cases/project/list_projects.py` and
  `backend/app/use_cases/project/create_project.py` ŌĆö both exist;
  no backend change required by this story.
* The empty-state copy includes `${user.first_name}` derived from
  `state.user.display_name` (J-001's projection). DESIGN owns the
  first-name extraction (`state.user.display_name.split(" ")[0]` is
  the simplest viable form).
