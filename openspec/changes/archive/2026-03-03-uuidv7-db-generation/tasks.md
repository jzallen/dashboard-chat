- [ ] 1.1 Add `uuidv7()` custom function registration in `backend/app/database.py`: Add a `"connect"` event listener on `engine.sync_engine` that fires when the database URL contains `sqlite`. The listener calls `dbapi_connection.create_function("uuidv7", 0, lambda: str(uuid7()))` using `uuid_utils.uuid7`. Import `uuid_utils` only inside the listener or at module level behind the sqlite guard.
- [ ] 1.2 Add deterministic `uuidv7()` factory function in `backend/tests/uuidv7_fixtures.py`: Add a `make_test_uuidv7()` function that returns a zero-arg callable. The callable uses a closure with an incrementing counter to produce IDs in the `019515a0-f{counter:03x}-7000-8000-{counter:012x}` format. Each call to `make_test_uuidv7()` creates a fresh counter starting at 0.
- [ ] 1.3 Register deterministic `uuidv7()` in `backend/tests/conftest.py`: On the test engine, add a `"connect"` event listener that calls `dbapi_connection.create_function("uuidv7", 0, make_test_uuidv7())`. Import `make_test_uuidv7` from `tests.uuidv7_fixtures`. This ensures every test's `CREATE TABLE ... DEFAULT (uuidv7())` succeeds, and any omitted IDs get deterministic values.
- [ ] 1.4 Update `backend/scripts/setup_dev.py` if it creates its own SQLite engine: Ensure the dev setup script also registers the `uuidv7()` function on its SQLite connections via the same event pattern. Verify the script uses the shared engine from `database.py` (in which case 1.1 covers it) or add a standalone registration.
